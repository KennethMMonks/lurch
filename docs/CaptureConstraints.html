    CaptureConstraints

    A capture constraint (singular) is defined in the CaptureConstraint class, but a set of capture constraints (plural) is defined here, in the CaptureConstraints class.

    Constructor

    new CaptureConstraints(…args)

    Create a new set of capture constraints by providing a list of objects to add to the set. Depending on what types of objects are provided, different actions are performed.

    • For any argument that is a CaptureConstraint, it is added to the list of capture constraints in this set as long as an identical one does not already appear on that list (judging by equality for capture constraints). We add it using the add() function.
    • For any argument that is a LogicConcept, it is treated as a pattern (as documented at the top of the Constraint module) and scanned to find all pairs of variables $u,v$ where at least one is a metavariable and $v$ appears free inside a binding that binds $u$. It then adds the corresponding capture constraint to this object. The pairs are found with the scan() function and the constraints added using the add() function.
    • For any argument that is a Constraint, we ignore its expression and just process its pattern, as in the previous bullet point.
    • Any other type of argument throws an error.

    Parameters

    • args Constraint | LogicConcept | CaptureConstraint <repeatable>

      the constraints to add to this list, as documented above

    • scan()

    scan(pattern, …boundopt)

    This method is for internal use only. It scans a pattern and finds all capture constraints in it and adds them to this class (using the add() function). A capture constraint $(b,f)$ is one in which either $b$ or $f$ is a metavariable (or both are) and $f$ occurs free inside a binding that binds $b$ in the given pattern.

    Parameters

    • pattern Expression

      the expression (typically containing metavariables) to scan for capture constraints

    • bound Symbol <optional>
      <repeatable>

      a list of variables bound above the pattern being scanned; this pattern is typically used only in recursive calls, and clients can ignore it

    toString() → {string}

    The string representation of a capture constraint set is simply the comma-separated list of string representations of its capture constraints, surrounded by curly brackets to suggest a set. For example, it might be "{(x,3),(y,K)}" or "{}".
